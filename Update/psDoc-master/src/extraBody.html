Getting Started

General information and steps to use this module for the first time.

Module Description

This module provides a turnkey solution for University of Wisconsin-Madison domain administrators to manage their domain's Office 365 accounts. It integrates the Domain Admin API, converting all available methods to PowerShell functions for simple execution from the console or in scripts. Functions can be added, removed, and updated as the API is modified. Additionally, this module provides the ability to easily add, save, and manage API connection parameters for later use.

Get Help

After importing the module, enter the function Get-O365Help to launch this documentation.


Add a Connection

You must add at least one API connection before most functions can be used. Use the function Add-O365Connection to enter the information provided to you by the DoIT Mail Team. If you choose to save this connection, it can be easily used in later sessions. After adding a connection, that connection is set as the current connection to be used. All functions in the module can now be used.

API Information & Credential Request


Update API Functions

After a connection is added, use the function Update-O365APIFunctions to generate PowerShell functions for all available API methods, as defined in the Domain Admin API Documentation. After updating, use the function Get-O365Help to get an updated version of this document.


View Available Connections

Use the function Get-O365Connections to view all the connections that are saved and in-memory.


Set Current Connection

Use the function Set-O365Connection to set a connection to be used as the current connection. This also clears the current connection if no connection is specified.


View Current Connection

Use the function Get-O365CurrentConnection to view the connection that is set as the current connection.


Remove Connections

Use the function Remove-O365Connection to remove a connection that is saved or in-memory.


Important Information

Useful information about the module.

File Locations

  • To find the path where the module is stored, use the function (Get-Module -ListAvailable WiscO365).path. These are static files that don't change when the functions are updated.

  • Files generated by the module are stored in %LOCALAPPDATA%\WindowsPowerShell\ModuleData\WiscO365.

Default Parameters

  • Default API Endpoint: https://wiscmail.wisc.edu/domainadmin.json

  • Default Domain: $Global:O365CurrentConnection.domain

Function Names

  • The PowerShell function names are different than the API method names in order to follow PowerShell convention, which specifies that function names must follow the format Verb-PrefixNoun.

  • For all functions, the prefix is O365.
    • Ex: addAllowedNetID becomes Add-O365AllowedNetID.

  • If the method originally had a prefix of MSOL, this is removed to avoid conflict with the official Office 365 PowerShell module from Microsoft.
    • Ex: editMsolUser becomes Edit-O365User.

  • Most verbs are identical to the original verb, but some have been changed if the original verb is not a "PowerShell-Approved" verb. However, the function has an alias with the original verb.
    • Ex: deleteServiceAccount becomes Remove-O365ServiceAccount, which has the alias Delete-O365ServiceAccount.

  • When updating the functions (Update-O365APIFunctions), the original name method name can be added as an alias to the function by using the switch includeAlias.


Connections

  • Saved connection information is stored in %LOCALAPPDATA%\WindowsPowerShell\ModuleData\WiscO365\Connections\connections.csv.

  • Saved passwords are stored in %LOCALAPPDATA%\WindowsPowerShell\ModuleData\WiscO365\Connections as XML files with the title synonymous to the connection ID. Saved passwords can only be used on the computer and by the user account on which it was originally created.

  • After importing the module, connection information is stored in Global variables.
    • $Global:O365Connections - Connection information for all connections, both saved and in-memory.
    • $Global:O365CurrentConnection - Connection information for the current connection.
    • $Global:O365Session - API session information.

  • For functions that require a "domain" parameter, the domain specified in the current connection will be used by default, unless a different domain is specified in the function.
